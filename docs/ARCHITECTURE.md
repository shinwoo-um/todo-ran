# 구조 설명

## 한 그림

```
┌──────── 브라우저 ─────────────────────────────────┐
│   화면 (React + Tailwind)                          │
│        ↓                                           │
│   훅(hook) ←── 전역 알림(할 일이 바뀌었어요 등)    │
│        ↓                                           │
│   저장 함수 모음 (브라우저 DB에 쓰기 + 보낼 일 쌓기)│
│        ↓                                           │
│   브라우저 DB (IndexedDB, Dexie 라이브러리 사용)   │
│        ↕                                           │
│   동기화 엔진 (서버로 보내기 / 서버에서 받기)       │
└────────────────────┬───────────────────────────────┘
                     ↓
              ┌── Supabase ──────────────┐
              │ Postgres DB + 접근 권한   │
              │ 로그인                    │
              │ 파일 저장소              │
              └───────────────────────────┘
```

## 층 구성

| 층 | 위치 | 하는 일 |
|---|---|---|
| 화면 | `src/app/`, `src/components/` | 페이지/컴포넌트, 사용자 입력 처리 |
| 훅(hook) | `src/hooks/` | 전역 알림을 듣고 화면을 자동으로 새로고침 |
| 저장 함수 | `src/lib/db/repo.ts` | 브라우저 DB 읽기/쓰기, 서버로 보낼 일 쌓기 |
| 동기화 | `src/lib/sync/` | 보내기/받기, 충돌 해결, 게스트→로그인 이관 |
| Supabase | `supabase/migrations/` | 실제 데이터 보관, 권한 정책, 로그인, 파일 |

## 핵심 결정

- **브라우저 우선** — 화면이 보여주는 데이터는 항상 브라우저 안의 DB에서 꺼낸다. 서버는 백그라운드로 같이 맞춰지기만 한다. 인터넷이 끊겨도 앱은 그대로 동작한다.
- **충돌은 마지막에 쓴 쪽이 이긴다** — 두 기기에서 같은 할 일을 거의 동시에 고치면, 시각(`updated_at`)이 더 늦은 쪽을 채택한다. 와이프 한 명이 쓰는 앱이라 충돌이 일어날 일이 거의 없다.
- **시각은 서버가 찍는다** — 브라우저 시계를 믿지 않는다. DB 트리거가 수정될 때마다 자동으로 현재 시각을 박는다.
- **id는 브라우저가 만든다** — `crypto.randomUUID()`로 새 id를 만들어 쓴다. 인터넷이 끊겨도 새 할 일을 만들 수 있다.
- **삭제는 표시만 한다** — 진짜 지우는 게 아니라 `deleted_at`에 시각을 적는다. 동기화가 꼬이지 않도록.
- **게스트가 가입하면** — 게스트 id로 쌓인 데이터의 주인을 실제 로그인 id로 한 번에 교체해서 서버로 올린다.

## 데이터 흐름 (할 일 추가)

```
화면 → 훅의 create() 호출
   → 저장 함수가 브라우저 DB에 쓰고 "보낼 일" 큐에 쌓는다
   → "할 일이 바뀌었어요" 전역 알림을 쏜다
   → 모든 페이지의 훅이 알림을 받고 화면을 새로 그린다
   → 잠시 후(200ms) 백그라운드에서 서버에 보낸다
```

## 품질 관리 자동화

- **husky + lint-staged** — 커밋할 때 바뀐 파일에만 ESLint/Prettier를 자동으로 돌린다
- **commitlint** — 커밋 메시지가 `feat: ...`, `fix: ...` 같은 정해진 형식을 따르도록 강제한다
- **Prettier + .editorconfig** — 들여쓰기, 줄 길이, 따옴표 같은 코드 모양을 통일한다
- **Vitest** — 핵심 로직을 자동으로 검증하는 테스트 (충돌 해결, 보낼 일 중복 제거, DB 읽기/쓰기, 게스트 이관 등 97개)
- **`npm run verify`** — 타입 검사 + 코드 점검 + 테스트를 한 번에 돌린다. 커밋 전에 실행 권장.
